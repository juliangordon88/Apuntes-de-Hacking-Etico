Una vez finalizada la fase de análisis durante la cual hemos descubierto una serie de vulnerabilidades de nuestro objetivo, deberemos encontrar la manera de explotar dichas vulnerabilidades. Para ello podemos hacer uso de una serie de motores de búsqueda
.

EXPLOIT-DB
Se trata de un fichero público de encargados de seguridad y 
pentesting
 que funciona mediante la aportación de cada uno de los usuarios que desean integrar una vulnerabilidad nueva a la base de datos. Es un proyecto desarrollado en conjunto con 
Offensive Security
 y cuenta con el apoyo de muchos especialistas, lo que hace que sea una de las principales fuentes a las que recurrir. El repositorio de 
Exploit Database
 está instalado en Linux y puede actualizarse mediante la sentencia 
apt install exploitdb
 para tenerlo disponible en el disco duro del entorno y listo para poder buscarlo mediante el script 
searchsploit. 

RAPID7-DB
La empresa de desarrollo de 
software
 para 
pentesting
 Rapid7 también cuenta con su propio repositorio de vulnerabilidades. De forma similar a exploit-db el repositorio nos proporciona información sobre la vulnerabilidad, su criticidad y gravedad, aunque no dispone de un repositorio de vulnerabilidades para poder utilizar ya que va implementado en sus aplicaciones, como por ejemplo Metasploit Framework, el cual veremos en este temario.
 

0DAY.TODAY  
Comparte muchas características a los dos repositorios que hemos presentado antes, sin embargo la diferencia es que además de contar con vulnerabilidades públicas, 0day.today permite realizar pagos para obtener 
exploits
 «día cero». En función de la gravedad de la vulnerabilidad y de la persona que lo venda el precio de este puede ir variando
